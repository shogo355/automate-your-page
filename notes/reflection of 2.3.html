What is a function (Dave calls them â€œprocedures")?

A function is like a penny crusher. It takes input(penny) and produce output(Souvenir Medal) as it defined(modeled). Except, a function can input many types of variables or strings (so the out put). For example, if you define a function convert_hours_to_munites(hours), it will simply multiply whatever input by 60.

What is the difference between making and using a function?

The function can be defined(made) by using def like below.

def convert_hours_to_minutes(hours):
minutes=hours*60
return minutes
Here, convert_hours_to_minutes is the name of the function and hours between the brackets is the input. Once you declared the function name and input, there are the lines what function does to the input and what to output(by using return).

To use a function, write the name of the function and actual value of input between the brackets. The value can be anything like variable or string (as far as the function can manage).
A function can be included in a variable or you could just print it like follows:
my_variable=convert_hours_to_minutes(5)
print convert_hours_to_minutes(5)

How do functions help programmers avoid repetition?

A function can be used unlimited times once you defined it. Therefore you do not need to write same codes again.

What happens if a function doesn't have a return statement?

return in Python means what to output. Whatever the function does, without output the answer of the function will be None.
